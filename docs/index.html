E. Dist Max
Hint 1
Think about what happens when you expand the absolute values in the Manhattan distance formula $|x_i - x_j| + |y_i - y_j|$. Each absolute value can be positive or negative — how many cases are there, and can you simplify them?
Hint 2
The key identity: $|x_i - x_j| + |y_i - y_j| = \max((x_i+y_i)-(x_j+y_j),\; (x_i-y_i)-(x_j-y_j),\; \text{negatives})$. This means the answer is $\max(\max(x+y) - \min(x+y),\; \max(x-y) - \min(x-y))$. Compute two arrays: $s_i = x_i + y_i$ and $d_i = x_i - y_i$.
Hint 3
Compute $s = [x+y \text{ for each point}]$ and $d = [x-y \text{ for each point}]$. The answer is $\max(\max(s)-\min(s),\; \max(d)-\min(d))$. This runs in $O(N)$ — no sorting needed.

Preparing Olympiad
Hint 1
Notice that $n \leq 15$. This is a very small constraint. Think about how many total subsets exist for $n$ elements and whether you can check every single one.
Hint 2
With $n \leq 15$, there are $2^{15} = 32768$ subsets — easily within time limits. Enumerate all subsets using a bitmask from $0$ to $2^n - 1$. For each subset, check whether it satisfies all three conditions.
Hint 3
For each bitmask, extract the selected elements and compute: (1) the count of selected problems (must be $\geq 2$), (2) the sum of difficulties (must be in $[l, r]$), and (3) the difference between max and min difficulty (must be $\geq x$). Count subsets passing all three.
Hint 4
In Python, loop for mask in range(1, 1 << n). Use bit & (1 << i) to check membership. Collect the selected difficulties into a list, then check len >= 2, l <= sum <= r, and max - min >= x. This runs in $O(2^n \cdot n)$ which is trivially fast.

Prerequisites

Bitmask / Bit Manipulation

Bitmasks represent sets as integers, where bit i being 1 means element i is in the set. Enables O(1) set operations (union, intersection, membership) and DP over subsets when n is small (≤ 20). Also useful for encoding states compactly.

Basic operations — Set bit i: mask |= (1 << i). Clear bit i: mask &= ~(1 << i). Check bit i: (mask >> i) & 1. Toggle: mask ^= (1 << i). Count bits: __builtin_popcount(mask).
Iterating over subsets — Enumerate all subsets of {0,...,n-1}: for mask in range(1 << n). Enumerate all subsets of a mask: sub = mask; while sub > 0: sub = (sub-1) & mask.
Common pitfalls
  • Using bitmask for n > 25 — 2^25 = 33M states, borderline
  • 1 << n overflows 32-bit int for n ≥ 31 — use 1LL << n
  • Confusing | (OR/union) with & (AND/intersection)
  • Off-by-one in bit indexing — bit 0 is the least significant

Brute Force / Complete Search

Try all possibilities. It's the first approach to consider — if the search space is small enough, brute force is the simplest correct solution. Key skill: estimating whether brute force will pass within time limits, and pruning to make it faster.

Estimating complexity — ~10^8 operations per second in C++, ~10^7 in Python. If your search space is smaller, brute force works. n ≤ 20: try 2^n subsets. n ≤ 10: try n! permutations. n ≤ 40: meet in the middle.
Iterative enumeration — Nested loops to enumerate all combinations. For subsets: iterate bitmasks 0 to 2^n-1. For pairs: double loop. For triples: triple loop with pruning.
Common pitfalls
  • Forgetting to prune — brute force without pruning is often too slow
  • Miscounting complexity — n=25 is NOT feasible for 2^n in Python
  • Not considering brute force at all — it's often the intended solution for small n
  • Generating invalid states instead of pruning them early

Mere Array
Hint 1
Think about which elements can actually participate in swaps. The condition involves $\gcd$ equaling the global minimum $m = \min(a)$. What must be true about $a_i$ for $\gcd(a_i, m) = m$ to hold?
Hint 2
An element can be swapped with $m$ if and only if it is divisible by $m$. By using $m$ as an intermediary, any two elements divisible by $m$ can be rearranged freely. Elements not divisible by $m$ can never move — they must already be in their correct sorted position.
Hint 3
Algorithm: find $m = \min(a)$. Build the sorted version of the array. For every index $i$, if $a[i] \% m \neq 0$, check that $a[i]$ equals the sorted value at that position. If all such elements match, print YES; otherwise NO. Runs in $O(n \log n)$ for the sort.

Number Theory

GCD and LCM — GCD(a,b) using Euclidean algorithm: GCD(a,b) = GCD(b, a%b). LCM(a,b) = a/GCD(a,b) * b (divide first to avoid overflow).

Hint 1
Simulating every copy naively will be too slow — strings can grow huge through repeated copy-and-append cycles. Think about working backwards from the final state: what was the last query that set the server's string?
Hint 2
Trace the chain in reverse: the last type-3 query tells you which PC wrote to the server. That PC's string is whatever it copied from the server (type-1) plus all appends (type-2) after that copy. Recurse: what was the server's string at the time of that type-1 copy? This gives you a chain of segments to collect.
Hint 3
Walk backwards collecting append segments: find the last type-3 (say PC $p$ at time $t$), then find PC $p$'s last type-1 before $t$ (say time $t'$), gather all type-2 appends to $p$ between $t'$ and $t$, then repeat from $t'$. Reverse the collected segments at the end and concatenate. Each query and character is visited at most once, so this runs in $O(Q + \sum|s|)$.

Prerequisites

Dynamic Programming

DP solves problems by breaking them into overlapping subproblems and storing results to avoid recomputation. The two key requirements: optimal substructure (optimal solution contains optimal sub-solutions) and overlapping subproblems (same subproblem computed multiple times). At CF 1200-1700, you need classical DP patterns and the ability to define states and transitions.

State definition — The hardest part of DP. Ask: what parameters uniquely describe a subproblem? Common states: (index, remaining capacity), (position, mask), (left, right). The number of distinct states determines memory and time.
Transitions — The recurrence relation: how does dp[state] relate to smaller states? Example: dp[i] = max(dp[i-1], dp[i-2] + val[i]) for house robber. Getting transitions right is the core skill.
Common pitfalls
  • Defining too many state parameters → MLE/TLE. Simplify the state.
  • Forgetting base cases → wrong answer or infinite recursion
  • Wrong transition order in bottom-up → using values not yet computed
  • Not considering 'do nothing' as a valid transition

Greedy Algorithms

Greedy makes the locally optimal choice at each step, hoping it leads to a globally optimal solution. It works when the problem has the greedy choice property (local optimum leads to global optimum) and optimal substructure. Faster than DP but only applicable when you can prove (or trust) the greedy strategy works.

The greedy strategy — At each step, pick the option that looks best right now without considering future consequences. If the problem has the right structure, this gives the optimal answer.
Common pitfalls
  • Assuming greedy works without verification — always test edge cases
  • Wrong sorting criterion — the key to greedy is sorting by the RIGHT thing
  • Greedy on examples ≠ greedy in general — find counterexamples
  • Not handling ties in sorting — tiebreakers matter
